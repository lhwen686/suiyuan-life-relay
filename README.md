# 髓缘大爱，生命接力

“髓缘大爱，生命接力”造血干细胞捐献志愿服务项目中文单页展示网站，面向项目启动会、会议投屏和手机端浏览。

## 本地运行

首次运行先安装依赖：

```bash
npm install
```

启动开发服务器：

```bash
npm run dev
```

默认地址通常为：

```text
http://localhost:5173
```

如果现场需要固定局域网地址，可使用：

```bash
npm run dev -- --host 0.0.0.0
```

## 构建与预览

生产构建：

```bash
npm run build
```

构建产物输出到 `dist/`。

本地预览构建产物：

```bash
npm run preview
```

## 部署方式

本项目是纯前端静态站点，不需要后端、数据库或登录系统。部署时将 `dist/` 目录发布到静态托管服务即可，例如：

- GitHub Pages
- Vercel
- Netlify
- 学校或项目组提供的静态 Web 服务器

如果部署到子路径，请根据托管平台要求调整 Vite 的 `base` 配置。

## 现场展示备用方案

建议会前至少准备三种访问方式：

- 主方案：提前运行 `npm run dev` 或 `npm run preview`，使用浏览器打开投屏地址。
- 备用方案一：提前运行 `npm run build`，确认 `dist/` 产物可用；现场网络不稳定时用本机 `npm run preview` 展示。
- 备用方案二：会前导出浏览器全页 PDF 或截图，投屏设备无法运行 Node.js 时使用静态图片或 PDF 讲解。
- 备用方案三：将 `dist/` 目录提前部署到线上静态地址，并准备本机离线预览作为兜底。

现场建议：

- 使用 1920×1080 分辨率投屏。
- 浏览器缩放保持 100%，必要时使用全屏模式。
- 会前点击顶部导航和时间轴月份卡片，确认投屏设备触控板或遥控翻页可用。
- 保持本页面不填写、不提交任何个人敏感信息。

## 内容维护

页面核心内容集中在 `src/data/project.ts`。后续如需调整项目名称、核心数字、时间轴、入库流程、团队分工或会后行动清单，优先修改该文件。

页面仅保留“志愿者分组意向填写”的占位按钮，不实现表单提交，也不收集身份证号、健康信息、入库报名信息等敏感信息。
